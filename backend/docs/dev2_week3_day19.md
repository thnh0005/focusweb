# Dev2 Week 3 Day 19 - Document Extraction

Day 19 adds deterministic document extraction for uploaded study documents without
calling an AI provider. The existing `StudyDocument` table is reused; extracted
plain text stays in `extracted_text`, processing state stays in `status`, and
parser details are stored under `metadata.extraction`.

Supported formats:

- PDF: uses `pypdf` when installed and falls back to literal text extraction for
  simple PDFs. Encrypted PDFs are rejected.
- DOCX: uses ZIP and XML parsing to extract paragraphs and table cells.
- TXT: decodes UTF-8/UTF-8 BOM/CP1258 and rejects binary-looking content.

Safety rules:

- No binary file persistence was added.
- No AI summaries or flashcards are generated by the extraction service.
- Metadata stores aggregate parser information, checksum, page map offsets,
  counts, warnings, and safe error codes only.
- Extracted content is not copied into metadata or logs.
- Upload size and extracted character limits are configurable in Django settings.

Async extraction is available through `extract_document_text`, but the current
upload flow still extracts synchronously because the model does not persist a
source file for later workers.
